# ModerBox 视频分析功能设计方案

## 1. 需求分析

### 1.1 核心功能需求

用户需要一个桌面工具，能够：

1. **读取本地视频文件** - 支持常见视频格式（mp4, avi, mkv, mov 等）
2. **语音转写** - 提取音频，使用 Whisper 模型识别语音
3. **视觉分析** - 使用支持图像输入的大模型分析视频每一帧/关键帧
4. **文案整理** - 将语音转写 + 视觉分析的结果整合成完整视频文案

### 1.2 关键约束

- **三个独立的 API Gateway**：语音转写、视觉分析、文案整理可能使用不同的 API 服务提供商
- **GUI 配置友好**：需要在设置界面支持分别配置三个服务的 API
- **跨平台**：继承 ModerBox 的 Avalonia 跨平台特性

## 3. 三个服务的独立配置设计

### 3.1 设计原则

**每个服务独立配置，包含：**

- API Endpoint（支持不同 Gateway）
- API Key
- 模型选择
- 高级选项（超时、并发、温度等）

### 3.4 模型选择自定义支持

**需求背景**：由于三个服务可能使用不同的 API Gateway（如 OpenAI、硅基流动、阿里云、Claude 等），每个渠道支持的模型各不相同，因此模型选择必须支持自定义输入，而不仅仅是预设下拉列表。

**UI 设计**：

```
┌────────────────────────────────────────────────────────────┐
│  模型:  [gpt-4o    ▼]  [或输入自定义模型名称...]          │
│         └────────────────┘  └────────────────────────────┘   │
│         下拉预设列表        自定义输入框                     │
└────────────────────────────────────────────────────────────┘
```

**交互逻辑**：
1. 用户可以从下拉列表中选择预设模型（如 gpt-4o、gpt-4o-mini、claude-3-opus 等）
2. 用户也可以在下拉框中直接输入自定义模型名称
3. 当用户输入的内容不在预设列表中时，将其作为自定义模型使用
4. 系统将直接使用用户输入的模型名称调用 API，不做校验（因为不同 Gateway 的模型命名规则不同）

**预设模型列表建议**：

| 服务 | 预设模型选项 |
|------|-------------|
| 语音转写 | whisper-1 |
| 视觉分析 | gpt-4o, gpt-4o-mini, claude-3-opus, claude-3-sonnet, qwen-vl-plus |
| 文案整理 | gpt-4o, gpt-4o-mini, o1-mini, claude-3-sonnet, qwen-turbo |

## 4. 处理流程设计

### 4.1 阶段划分

```
┌─────────────────────────────────────────────────────────────────────────┐
│  阶段 0: 初始化                                                          │
│  - 验证所有启用的服务配置                                                │
│  - 创建临时工作目录                                                     │
│  - 获取视频元信息                                                       │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  阶段 1: 媒体提取 (可并行)                                              │
│  ┌──────────────────┐      ┌──────────────────┐                       │
│  │  1a. 提取视频帧   │      │  1b. 提取音频    │                       │
│  │  - 按间隔采样    │      │  - 转码为 WAV    │                       │
│  │  - 或关键帧      │      │  - 16kHz, mono   │                       │
│  └──────────────────┘      └──────────────────┘                       │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  阶段 2: 语音转写 (Whisper API)                                        │
│  - 上传音频文件到 Gateway 1                                            │
│  - 获取带时间戳的转写文本                                               │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  阶段 3: 视觉分析 (Vision API) - 并行处理                               │
│  - 并发调用 Gateway 2 分析每一帧                                        │
│  - 限制并发数 (避免 API 限流)                                          │
│  - 返回每帧的画面描述                                                   │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  阶段 4: 文案整理 (Summary API)                                        │
│  - 将语音转写 + 视觉描述发送给 Gateway 3                               │
│  - 生成最终文案                                                         │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  阶段 5: 输出与清理                                                     │
│  - 保存文案到文件                                                       │
│  - 清理临时文件                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 4.2 进度报告机制

每个阶段都应报告进度，格式：

```csharp
class AnalysisProgress {
    Stage:          // 当前阶段 (Analyzing/Extracting/Transcribing/AnalyzingFrames/Summarizing/Completed)
    StageProgress: // 当前阶段内进度 0-100
    OverallProgress:// 整体进度 0-100
    Message:        // 描述性消息
    CurrentItem:    // 当前处理项 (如 "处理帧 5/50")
}
```

## 5. 配置自动存储

### 5.1 存储机制

**存储位置**：
- Windows: `%APPDATA%\ModerBox\video-analysis.json`
- macOS: `~/.config/ModerBox/video-analysis.json`
- Linux: `~/.config/ModerBox/video-analysis.json`

**存储格式**：JSON 格式，示例见 3.3 节

### 5.2 自动保存策略

1. **即时保存**：用户修改配置后自动保存，无需手动点击"保存"按钮
2. **加密存储**：API Key 等敏感信息使用 DPAPI（Windows）/Keychain（macOS）/libsecret（Linux）加密存储
3. **配置迁移**：新版本发布时自动迁移旧配置文件

### 5.3 配置验证

- **启动时验证**：检查配置文件完整性，缺失字段使用默认值
- **连接测试**：提供"测试连接"按钮，用户可手动验证 API 配置是否正确
- **错误提示**：配置错误时显示友好提示，不影响程序启动

支持变量替换，简化批量输出命名：

| 变量 | 说明 | 示例 |
|------|------|------|
| `{filename}` | 原始视频文件名（不含扩展名） | `video_001` |
| `{index}` | 序号（从 1 开始） | `1`, `2`, `3` |
| `{date}` | 处理日期 (yyyy-MM-dd) | `2025-03-12` |

- **默认模板**：`{filename}_文案` → `video1_文案.md`
- **用户可自定义**：如 `{filename}_{date}` → `video1_2025-03-12.md`

### 6.3 批量处理流程

                                    │

- **失败重试**：可配置失败后重试次数

## 7. 技术选型

### 7.1 视频处理

| 方案 | 优点 | 缺点 |
|------|------|------|
| **FFMpegCore** | 纯 .NET 封装，功能完整，跨平台 | 需要 FFmpeg 二进制 |
| Xabe.FFmpeg | API 友好 | 维护频率一般 |
| 直接调用 FFmpeg 进程 | 灵活 | 需要处理进程生命周期 |

**推荐**：FFMpegCore，自动下载 FFmpeg，配置简单

### 7.2 API 调用

| 服务 | 推荐 API 端点 | 备注 |
|------|---------------|------|
| 语音转写 | `/v1/audio/transcriptions` | OpenAI Whisper |
| 视觉分析 | `/v1/responses` + input_image | OpenAI Response API |
| 文案整理 | `/v1/responses` | OpenAI Response API |

**注意**：Response API 支持多种 provider，关键是配置正确的 endpoint

### 7.3 依赖项

| 包 | 用途 |
|----|------|
| FFMpegCore | 视频/音频处理 |
| System.Net.Http.Json | HTTP 调用 |
| Avalonia | UI 框架 |
| ReactiveUI | MVVM |

## 9. 错误处理策略

| 阶段 | 错误类型 | 处理策略 |
|------|----------|----------|
| 媒体提取 | 文件格式不支持 | 提示用户，支持格式列表 |
| 媒体提取 | FFmpeg 缺失 | 提示下载/配置路径 |
| API 调用 | 认证失败 | 提示检查 API Key |
| API 调用 | 限流 | 自动重试 + 退避 |
| API 调用 | 超时 | 可配置超时，允许跳过失败帧 |
| 整体 | 用户取消 | 清理临时文件，正常退出 |
