# 电缆走向自动化绘制

## 功能介绍

**电缆走向绘制** 模块提供自动化的电缆路由规划和绘制功能。根据用户提供的配置文件（JSON 格式），程序能够：

1. **智能路径规划**：利用 Dijkstra 算法计算观测点间的最短路径
2. **自动连线绘制**：
   - 直线连接：观测点 ↔ 观测点、同 pair 的穿管点
   - L 型连接：start/end/pass → observation
3. **碰撞检测**：在终点附近自动放置业务统计表格，避免与已有元素重叠
4. **高质量输出**：基于 SkiaSharp 生成 PNG/JPG 格式的矢量级图像

## 快速开始

### 1. 创建示例配置

在应用中打开"电缆走向绘制"标签页，点击 **"创建示例"** 按钮，即可生成示例配置文件。

### 2. 编辑配置文件

编辑生成的 JSON 配置文件，定义电缆网络中的所有点位和业务统计数据。

### 3. 执行绘制

- **配置文件**：必选，需先创建或选择配置文件
- **底图路径**：可选，若指定则覆盖配置文件中的设置
- **输出路径**：可选，若指定则覆盖配置文件中的设置

点击 **"开始绘制"** 开始处理。程序会实时输出执行日志，最后自动打开输出目录。

#### 顶级属性

| 字段 | 类型 | 必需 | 说明 |

#### points 数组元素（RoutePoint）

| 字段 | 类型 | 必需 | 说明 |

| 类型 | 说明 | 绘制颜色 | 特殊说明 |

| 字段 | 类型 | 必需 | 说明 |

| `count` | integer | 是 | 数量 |

## 路径规划算法

### 何时使用穿管点（Pass）

穿管点用于标记电缆需要通过管道或穿过障碍物的地点。当配置中包含任何 `Pass` 类型的点位时，路由规划会自动激活穿管模式。

### Pair 概念说明

`pair` 是穿管点的 **配对标识符**，用于指定哪两个穿管点属于同一条穿管线路。

**示例解析：**
```json
{
  "id": "P1A",
  "type": "Pass",
  "x": 300,
  "y": 300,
  "pair": "P1"  // 穿管线路编号
},
{
  "id": "P1B",
  "type": "Pass",
  "x": 500,
  "y": 300,
  "pair": "P1"  // 同一穿管线路编号
}
```

- **id**（点位标识）：全局唯一，用于区分每个点位。P1A 和 P1B 是不同的点位
- **pair**（配对标识）：用于关联同一条穿管线路。都设为 "P1" 表示这两个点是电缆穿过管道的入口和出口

**更多示例：**
```json
{
  "id": "P1A", "type": "Pass", "x": 100, "y": 100, "pair": "P1"  // 第1条穿管线路，入口
},
{
  "id": "P1B", "type": "Pass", "x": 200, "y": 100, "pair": "P1"  // 第1条穿管线路，出口
},
{
  "id": "P2A", "type": "Pass", "x": 300, "y": 200, "pair": "P2"  // 第2条穿管线路，入口
},
{
  "id": "P2B", "type": "Pass", "x": 400, "y": 200, "pair": "P2"  // 第2条穿管线路，出口
}
```

### 总体流程

1. **start 到最近的 observation**
2. **observation 图最短路**（使用 Dijkstra 算法）→ 某个 observation
3. **observation 到最近的 pass**（自动选择离当前点最近的穿管点）
4. **pass A 到 pass B**（同 pair，直线连接）
5. **pass 到最近的 observation**（朝向 end）
6. **observation 图最短路** → 离 end 最近的 observation
7. **observation 到 end**

### 多穿管点选择机制

当配置中有多条穿管线路时（多个 pair），程序在第 3 步会 **自动选择距离当前 observation 最近的 Pass 点**，以此确定使用哪一对穿管点。

**示例：**
假设有以下配置：
```json
{
  "id": "O1", "type": "Observation", "x": 200, "y": 200
},
{
  "id": "P1A", "type": "Pass", "x": 300, "y": 300, "pair": "P1"  // 线路1：向北穿管
},
{
  "id": "P1B", "type": "Pass", "x": 500, "y": 300, "pair": "P1"
},
{
  "id": "P2A", "type": "Pass", "x": 310, "y": 305, "pair": "P2"  // 线路2：向东穿管
},
{
  "id": "P2B", "type": "Pass", "x": 510, "y": 305, "pair": "P2"
}
```

当电缆从 O1 出发需要穿管时：
- 距 O1 最近的 Pass 点是 P2A（距离约 120 像素）
- 所以程序会选择 **pair="P2" 的穿管线路**
- 电缆会通过 P2A → P2B 穿管，而不是 P1A → P1B

这样可以 **优先使用距离最短的穿管线路**，符合实际工程中的最优化原则。

### 选择算法总结

```
对每条 observation 到 pass 的连接：
  1. 获取所有 Pass 点
  2. 计算当前点到每个 Pass 点的距离
  3. 选择距离最小的 Pass 点（假设为 P_nearest）
  4. 根据 P_nearest.pair 查找其配对点
  5. 绘制从 P_nearest 到其配对点的穿管段
```

### 连接规则

- **observation ↔ observation**：直线连接
- **同 pair 的 pass 点**：直线连接
- **start/end/pass → observation**：L 型正交连接

## 正交修正机制

当需要生成 L 型连接时，程序会在 observation 集合中搜索满足以下条件的中间拐点：

- `(x == A.x and y == B.y)`
- 或 `(y == A.y and x == B.x)`

如果存在多个，选择距离最短的；如果不存在，使用默认 L 型路径（先水平后垂直）。

## 常见问题

### Q: 什么时候需要使用穿管点（Pass）？

A: 当电缆需要通过管道、穿过墙壁或其他障碍物时。如果没有穿管需求，可以直接使用 observation 点来表示路由点。

### Q: pair 字段有什么作用？

A: `pair` 用于关联两个穿管点（入口和出口），表示同一条穿管线路。程序会：
1. 根据 pair 值自动查找配对的穿管点
2. 在这两个点之间绘制直线（代表穿管段）
3. 从配对的出口点继续规划后续路由

**重要**：同一个 pair 值必须恰好对应两个 Pass 类型的点位。

### Q: pair 可以重复吗？

A: 可以。如果有多条穿管线路，每条都应该有唯一的 pair 值。例如：
- 线路1：P1A 和 P1B 的 pair 都是 "P1"
- 线路2：P2A 和 P2B 的 pair 都是 "P2"

### Q: id 和 pair 的区别是什么？

A: 
- **id**：点位的全局唯一标识，每个点位必须有不同的 id
- **pair**：穿管点的配对标识，相同 pair 值的两个点会被视为一对。只有 Pass 类型的点才应该配置 pair

### Q: 不设置 pair 会怎样？

A: 如果 Pass 点没有 pair 值（或为空），程序会将其视为普通的路由点，不会执行穿管配对逻辑。

### Q: 有多条穿管线路时，程序怎么选择用哪一条？

A: 程序会 **自动选择距离当前 observation 最近的 Pass 点**，以此确定使用哪条穿管线路。
- 计算当前点到所有 Pass 点的距离
- 选择最近的 Pass 点
- 根据该点的 pair 值找到其配对点
- 通过这对点进行穿管

这样可以保证选择距离最短、最优的穿管路线。

### Q: 如果有多条平行的穿管线路，如何确保选择正确的方向？

A: 配置穿管点时要遵循以下原则：
- **地理位置**：将穿管点放在实际穿管的位置
- **方向性**：穿管点对应的方向应该指向最终目的地
- **距离优化**：更近的穿管线路会被优先选择

例如，如果有北向穿管和东向穿管两条线路，程序会根据 observation 点的位置自动选择距离最近的线路。

### Q: 配置文件中相对路径如何处理？

A: 相对路径会以配置文件所在目录为基准进行解析。在 GUI 中指定的路径优先级更高，会覆盖配置文件中的设置。

### Q: 输出图像尺寸如何确定？

A: 输出图像尺寸由底图决定。若底图不存在，程序会创建 900×700 的空白画布。

### Q: 表格位置冲突如何解决？

A: 程序会按以下优先级尝试放置表格：右上 → 右下 → 左上 → 左下 → 上 → 下

### Q: 支持哪些输出格式？

A: 基于文件扩展名自动识别，支持 PNG、JPG/JPEG、BMP、WebP。推荐使用 PNG 获得最佳质量。

## 示例工作流

### 场景：规划局域网布线

1. **获取底图**：拍照或扫描建筑平面图
2. **标注点位**：在平面图上标记各个布线点
3. **手工测量**：用标尺或激光测距工具获得坐标
4. **创建配置**：在程序中点击"创建示例"
5. **填写数据**：编辑配置文件，输入所有点位坐标
6. **运行绘制**：启动绘制过程
7. **验证结果**：检查生成的走向图是否符合实际需求

## 技术细节

### 图算法

- **图类型**：观测点间构成无向完全图
- **最短路径**：Dijkstra 算法，时间复杂度 O((V+E)logV)
- **距离度量**：欧氏距离

### 碰撞检测

使用轴对齐边界框（AABB）进行快速碰撞检测，支持位置优先级排序。

### 图像渲染

基于 **SkiaSharp** 库，提供：
- 抗锯齿线条绘制
- 文字描边效果
- 高质量导出
