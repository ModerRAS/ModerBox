# 电缆走向自动化绘制

## 功能介绍

**电缆走向绘制** 模块提供自动化的电缆路由规划和绘制功能。根据用户提供的配置文件（JSON 格式），程序能够：

1. **智能路径规划**：利用 Dijkstra 算法计算观测点网络间的最短路径
2. **电缆沟网络**：观测点之间只有水平或垂直排列时才会直接相连，形成电缆沟网络
3. **自动连线绘制**：
   - 直线连接：观测点 ↔ 观测点（仅水平/垂直）、同 pair 的穿管点
   - L 型连接：start/end/pass → 电缆沟 → observation
4. **碰撞检测**：在终点附近自动放置业务统计表格，避免与已有元素重叠
5. **高质量输出**：基于 SkiaSharp 生成 PNG/JPG 格式的矢量级图像
6. **路由记录**：自动生成经过观测点的文本记录文件

## 快速开始

### 1. 创建示例配置

在应用中打开"电缆走向绘制"标签页，点击 **"创建示例"** 按钮，即可生成示例配置文件。

### 2. 编辑配置文件

编辑生成的 JSON 配置文件，定义电缆网络中的所有点位和业务统计数据。

### 3. 执行绘制

- **配置文件**：必选，需先创建或选择配置文件
- **底图路径**：可选，若指定则覆盖配置文件中的设置
- **输出路径**：可选，若指定则覆盖配置文件中的设置

点击 **"开始绘制"** 开始处理。程序会实时输出执行日志，最后自动打开输出目录。

## 路径规划算法

### 电缆沟网络机制

**核心变化**：观测点之间不再是无向完全图，而是**电缆沟网络**：
- 只有**水平或垂直排列**的观测点才直接相连（角度容差 5°）
- 外部点（Start、End、Pass）通过**入沟/出沟**机制连接到电缆沟网络
- 入沟：外部点垂直投影到最近的电缆沟线段
- 出沟：从电缆沟线段垂直移动到目标点

### 何时使用穿管点（Pass）

穿管点用于标记电缆需要通过管道或穿过障碍物的地点。当配置中包含任何 `Pass` 类型的点位时，路由规划会自动激活穿管模式。

### Pair 概念说明

`pair` 是穿管点的 **配对标识符**，用于指定哪两个穿管点属于同一条穿管线路。

**示例解析：**
```json
{
  "id": "P1A",
  "type": "Pass",
  "x": 300,
  "y": 300,
  "pair": "P1"  // 穿管线路编号
},
{
  "id": "P1B",
  "type": "Pass",
  "x": 500,
  "y": 300,
  "pair": "P1"  // 同一穿管线路编号
}
```

- **id**（点位标识）：全局唯一，用于区分每个点位。P1A 和 P1B 是不同的点位
- **pair**（配对标识）：用于关联同一条穿管线路。都设为 "P1" 表示这两个点是电缆穿过管道的入口和出口

**更多示例：**
```json
{
  "id": "P1A", "type": "Pass", "x": 100, "y": 100, "pair": "P1"  // 第1条穿管线路，入口
},
{
  "id": "P1B", "type": "Pass", "x": 200, "y": 100, "pair": "P1"  // 第1条穿管线路，出口
},
{
  "id": "P2A", "type": "Pass", "x": 300, "y": 200, "pair": "P2"  // 第2条穿管线路，入口
},
{
  "id": "P2B", "type": "Pass", "x": 400, "y": 200, "pair": "P2"  // 第2条穿管线路，出口
}
```

### 总体流程

1. **start 入沟**：Start → 垂足点 → 电缆沟上的观测点
2. **电缆沟网络最短路**：Dijkstra 算法 → 目标电缆沟
3. **出沟到 PassA**：电缆沟 → 垂足点 → PassA
4. **PassA → PassB**：同 pair，直线连接
5. **PassB 入沟**：PassB → 垂足点 → 电缆沟上的观测点
6. **电缆沟网络最短路**：Dijkstra 算法 → 终点电缆沟
7. **出沟到 End**：电缆沟 → 垂足点 → End

### 连接规则

- **观测点 ↔ 观测点**：仅水平或垂直排列时才直线连接（构成电缆沟）
- **同 pair 的 pass 点**：直线连接
- **外部点（start/end/pass）→ 观测点**：通过入沟/出沟机制连接

## 正交修正机制

当需要生成 L 型连接时，程序会在 observation 集合中搜索满足以下条件的中间拐点：

- `(x == A.x and y == B.y)`
- 或 `(y == A.y and x == B.x)`

如果存在多个，选择距离最短的；如果不存在，使用默认 L 型路径（先水平后垂直）。

## 常见问题

### Q: 观测点需要按特定方式排列吗？

A: **是的**。电缆沟网络要求观测点之间只有水平或垂直排列时才会直接相连。角度容差为 5°，即与水平或垂直夹角小于 5° 的点会被视为同一条直线。

这意味着：
- 观测点应按网格或直线排列
- 倾斜的观测点之间不会直接相连，需要经过其他观测点中转

### Q: 外部点如何连接到观测点网络？

A: 使用**入沟/出沟机制**：
- **入沟**：从外部点（Start/End/Pass）向最近的电缆沟线段做垂线，垂足点即为入沟点
- **出沟**：从电缆沟上的点垂直移动到目标点
- 路由中会生成临时的"垂足点"（标记为 `_foot_`），用于计算最短路径

### Q: 外部点如何选择连接到电缆沟的哪个位置？

A: 程序会：
1. 找到离外部点最近的电缆沟线段
2. 计算垂直投影点（垂足）
3. 根据方向提示选择电缆沟的两个端点中更优的一个
4. 返回入沟点和总距离

A: 当电缆需要通过管道、穿过墙壁或其他障碍物时。如果没有穿管需求，可以直接使用 observation 点来表示路由点。

### Q: 什么时候需要使用穿管点（Pass）？

A: 当电缆需要通过管道、穿过墙壁或其他障碍物时。如果没有穿管需求，可以直接使用 observation 点来表示路由点。

### Q: pair 字段有什么作用？

A: `pair` 用于关联两个穿管点（入口和出口），表示同一条穿管线路。程序会：
1. 根据 pair 值自动查找配对的穿管点
2. 在这两个点之间绘制直线（代表穿管段）
3. 从配对的出口点继续规划后续路由

**重要**：同一个 pair 值必须恰好对应两个 Pass 类型的点位。

### Q: pair 可以重复吗？

A: 可以。如果有多条穿管线路，每条都应该有唯一的 pair 值。例如：
- 线路1：P1A 和 P1B 的 pair 都是 "P1"
- 线路2：P2A 和 P2B 的 pair 都是 "P2"

### Q: id 和 pair 的区别是什么？

A: 
- **id**：点位的全局唯一标识，每个点位必须有不同的 id
- **pair**：穿管点的配对标识，相同 pair 值的两个点会被视为一对。只有 Pass 类型的点才应该配置 pair

### Q: 不设置 pair 会怎样？

A: 如果 Pass 点没有 pair 值（或为空），程序会将其视为普通的路由点，不会执行穿管配对逻辑。

### Q: 有多条穿管线路时，程序怎么选择用哪一条？

A: 程序会**尝试正反向两条路线**，并选择总长度最短的一条：
- 路线1: Start → Obs → PassA → PassB → Obs → End
- 路线2: Start → Obs → PassB → PassA → Obs → End

### Q: 如果有多条平行的穿管线路，如何确保选择正确的方向？

A: 配置穿管点时要遵循以下原则：
- **地理位置**：将穿管点放在实际穿管的位置
- **方向性**：穿管点对应的方向应该指向最终目的地
- **程序选择**：会自动计算两条路线的总长度，选择较短的

### Q: 配置文件中相对路径如何处理？

A: 相对路径会以配置文件所在目录为基准进行解析。在 GUI 中指定的路径优先级更高，会覆盖配置文件中的设置。

### Q: 程序怎么确定是走穿管还是走观测点？怎么保证去不同房间一定走穿管？

A: **关键在于配置文件中的房间划分设计**：

#### 路由决策机制

程序遵循以下 7 步路由流程：

1. **Start 入沟**：Start → 垂足 → 电缆沟上的观测点
2. **电缆沟网络最短路**：使用 Dijkstra 在水平/垂直观测点网络中计算最短路径
3. **出沟到 PassA**：电缆沟 → 垂足 → PassA
4. **PassA → PassB**：配对穿管，同 pair 值，直线连接
5. **PassB 入沟**：PassB → 垂足 → 电缆沟上的观测点
6. **电缆沟网络最短路**：Dijkstra 计算到终点电缆沟
7. **出沟到 End**：电缆沟 → 垂足 → End

#### 保证跨房间走穿管的配置原则

要确保去不同房间**一定**走穿管，需要在配置时遵循：

**原则 1：按房间组织观测点**
- 同一房间的观测点应该水平或垂直排列（形成电缆沟）
- 不同房间的观测点应该物理上分开

**原则 2：穿管点作为房间之间的唯一通道**
- Pass 点应该放在房间边界处
- 配对的 Pass A 和 Pass B 分别对应不同房间

**原则 3：配置示例**
```json
{
  "observations": [
    {"id": "O1", "x": 100, "y": 100},    // 房间1，水平排列
    {"id": "O2", "x": 200, "y": 100},    // 房间1，水平排列
    {"id": "O3", "x": 500, "y": 100},    // 房间2，水平排列
    {"id": "O4", "x": 600, "y": 100}     // 房间2，水平排列
  ],
  "passes": [
    {"id": "P1A", "x": 280, "y": 80, "pair": "P1"},    // 房间1到房间2的通道
    {"id": "P1B", "x": 300, "y": 120, "pair": "P1"}    // 房间2对应的通道
  ]
}
```

#### 实际路由效果

当 Start 在房间 1，End 在房间 2 时：

```
Start → O1 → O2 → P1A → P1B → O3 → O4 → End
        └─ 电缆沟 ─┘    └─ 穿管 ┘   └─ 电缆沟 ─┘
```

- **Start 入沟**：Start 垂直投影到 O1-O2 电缆沟线段
- **O1 → O2**：电缆沟网络直线（水平）
- **O2 出沟到 P1A**：垂直移动到穿管入口
- **P1A → P1B**：穿过房间间的障碍物（直线）
- **P1B 入沟**：垂直投影到 O3-O4 电缆沟线段
- **O3 → O4**：电缆沟网络直线（水平）
- **O4 出沟到 End**：垂直移动到终点

#### 为什么不会跳过穿管？

因为：
1. 电缆沟网络只在水平/垂直的观测点之间相连
2. 不同房间的观测点之间没有直接连接（除非经过穿管点）
3. 程序会自动计算最短路径，确保经过穿管

### Q: 输出图像尺寸如何确定？

A: 输出图像尺寸由底图决定。若底图不存在，程序会创建 900×700 的空白画布。

### Q: 表格位置冲突如何解决？

A: 程序会按以下优先级尝试放置表格：右上 → 右下 → 左上 → 左下 → 上 → 下 → 四角 → 边缘中点

### Q: 支持哪些输出格式？

A: 基于文件扩展名自动识别，支持 PNG、JPG/JPEG、BMP、WebP。推荐使用 PNG 获得最佳质量。

### Q: 路由文本记录是什么？

A: 程序会自动生成一个 `.txt` 文件，包含：
- 完整路径（包含所有点）
- 经过的观测点（按顺序，排除垂足点）
- 观测点ID列表（纯列表，方便复制）

文件名与输出图片相同，只是扩展名不同（例如 `result.png` → `result.txt`）。

### Q: lineWidth 配置项的作用？

A: `lineWidth` 用于控制电缆路径线条的粗细（像素），默认值为 3。值越大线条越粗。

## 技术细节

### 电缆沟网络

- **图类型**：观测点间构成电缆沟网络
- **连接规则**：只有水平或垂直（角度容差 ≤5°）排列的观测点才会直接相连
- **最短路径**：Dijkstra 算法，时间复杂度 O((V+E)logV)
- **距离度量**：欧氏距离

### 入沟/出沟机制

外部点（Start、End、Pass）通过以下方式连接到电缆沟网络：
1. 找到最近的电缆沟线段
2. 计算垂直投影点（垂足）
3. 沿电缆沟移动到目标区域
4. 垂直移动到目标点

### 垂足点

路由中会自动生成临时的"垂足点"（标记为 `_foot_`），用于：
- 连接外部点到电缆沟网络
- 计算最短路径

这些点在图像中不会绘制，只用于路径计算。

### 碰撞检测

使用轴对齐边界框（AABB）进行快速碰撞检测，支持位置优先级排序。

### 图像渲染

基于 **SkiaSharp** 库，提供：
- 抗锯齿线条绘制
- 文字描边效果
- 高质量导出
